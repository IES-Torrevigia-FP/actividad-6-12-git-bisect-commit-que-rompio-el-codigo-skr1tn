git bisect sirve para encontrar el commit exacto donde se introduce un bug usando búsqueda binaria. Se marcan un estado “buggy” y uno “good”, y Git va probando puntos intermedios. Es útil en proyectos con muchos commits donde no se sabe cuándo apareció un fallo. Funciona mejor si el bug es reproducible y se puede comprobar claramente si está o no. Los tests automáticos y un historial limpio hacen el proceso mucho más rápido y fiable.
